## 4. Schema do JSON bundled

`taxonomy_with_embeddings.json` é um **array de 1331 objetos** com este shape:

```jsonc
[
  {
    "codigo": "4530703",
    "secao": "G",
    "divisao": "45",
    "grupo": "453",
    "classe": "45307",
    "denominacao": "Comércio a varejo de peças e acessórios novos para veículos automotores",
    "notas_explicativas": "Esta subclasse compreende:\n- o comércio a varejo de peças, partes e acessórios novos para veículos automotores\n- ...\nEsta subclasse não compreende:\n- o comércio a varejo de pneumáticos usados (45.30-7-05)",
    "exemplos_atividades": "- o comércio a varejo de peças, partes e acessórios novos para veículos automotores\n- o comércio a varejo de pneumáticos novos",
    "embedding": [0.012345, -0.045678, 0.001234, /* ... 1024 floats total ... */]
  }
]
```

**Tamanho estimado:** ~7MB com floats serializados em `.6f` (precisão preservada além do que Voyage retorna como significativo). Repo cresce de ~3MB para ~10MB — aceitável.

**Por que JSON e não Parquet/.npz:** legibilidade no review (diff humano de mudanças de denominação ou nota), e leitura simples sem dependência extra. Trade-off de ~3MB em disk é irrelevante.

**`taxonomy.json`** (sem embedding) tem o mesmo shape menos a key `embedding` — ~1.5MB. Existe para:
- Permitir regenerar embeddings sem reparsear XLS
- Servir de fonte canônica de texto se trocarmos modelo de embedding

## 5. Texto enviado ao embedding

```python
# scripts/embed_taxonomy.py
def render_doc(row: dict) -> str:
    parts = [row["denominacao"]]
    if row.get("notas_explicativas"):
        parts.append(row["notas_explicativas"])
    if row.get("exemplos_atividades"):
        parts.append(f"Atividades: {row['exemplos_atividades']}")
    text = ". ".join(parts)
    assert len(text) < 30000, f"texto excede limite Voyage para {row['codigo']}"
    return text
```

**Distinção document vs query:** Voyage-3 aceita `input_type="document"` (para o corpus indexado) e `input_type="query"` (para a busca). Documentação Voyage recomenda usar ambos para melhor recall em retrieval. Aplicamos:

- `embed_taxonomy.py` → `input_type="document"`
- `VoyageClient.embed_query()` (chamado em runtime) → `input_type="query"`
- `VoyageClient.embed_documents()` (helper genérico para batch) → `input_type="document"`

Sem truncamento ativo (todas as entradas reais cabem com folga em 32K tokens).

## 6. Migration 0005

Arquivo `backend/alembic/versions/0005_seed_cnae_taxonomy.py`:

```python
"""seed cnae taxonomy: ivfflat index

Revision ID: 0005
Revises: 0004
Create Date: 2026-05-12
"""

from alembic import op

revision = "0005"
down_revision = "0004"
branch_labels = None
depends_on = None


def upgrade() -> None:
    op.execute(
        """
        CREATE INDEX IF NOT EXISTS cnae_taxonomy_embedding_idx
        ON cnae_taxonomy
        USING ivfflat (embedding vector_cosine_ops)
        WITH (lists = 40);
        """
    )
    op.execute("ANALYZE cnae_taxonomy;")


def downgrade() -> None:
    op.execute("DROP INDEX IF EXISTS cnae_taxonomy_embedding_idx;")
```

**Observações:**
- Migration **não** insere dados (só índice). Loader é responsabilidade do script.
- `lists = 40` calibrado para 1331 rows (heurística pgvector: `lists ≈ √N` para N ≤ 1M).
- `ANALYZE` na tabela vazia é barato; loader chama `ANALYZE` de novo no final, gerando stats reais.
- `IF NOT EXISTS` torna `upgrade()` idempotente (rerun seguro).

## 7. Loader script — `scripts/load_cnae_taxonomy.py`

**Responsabilidades:**
1. Lê `backend/data/cnae_2.3/taxonomy_with_embeddings.json`
2. Valida count == 1331, todos embedding len == 1024
3. Conecta via `DATABASE_URL` (asyncpg direto, sem SQLAlchemy ORM — performance + simplicidade)
4. Bulk UPSERT via `INSERT … ON CONFLICT (codigo) DO UPDATE SET …`
5. Roda `ANALYZE cnae_taxonomy;` no final
6. Verifica `SELECT COUNT(*) FROM cnae_taxonomy = 1331`; exit 1 se diferente
7. Log: `Loaded 1331 CNAE subclasses in {elapsed}s`

**Estratégia de idempotência:** UPSERT por `codigo` permite:
- Rerun cego (mesmo input → mesmo estado, no-op nos rows)
- Trocar modelo de embedding (regerar JSON → rerun → todos embeddings substituídos)
- Adicionar/atualizar `notas_explicativas` sem `TRUNCATE`

**Esqueleto:**

```python
"""Load CNAE 2.3 taxonomy + embeddings into cnae_taxonomy table."""

import asyncio
import json
import os
import time
from pathlib import Path

import asyncpg

DATA_PATH = Path(__file__).parent.parent / "data" / "cnae_2.3" / "taxonomy_with_embeddings.json"

UPSERT_SQL = """
INSERT INTO cnae_taxonomy
    (codigo, secao, divisao, grupo, classe, denominacao,
     notas_explicativas, exemplos_atividades, embedding)
VALUES ($1, $2, $3, $4, $5, $6, $7, $8, $9::vector)
ON CONFLICT (codigo) DO UPDATE SET
    secao = EXCLUDED.secao,
    divisao = EXCLUDED.divisao,
    grupo = EXCLUDED.grupo,
    classe = EXCLUDED.classe,
    denominacao = EXCLUDED.denominacao,
    notas_explicativas = EXCLUDED.notas_explicativas,
    exemplos_atividades = EXCLUDED.exemplos_atividades,
    embedding = EXCLUDED.embedding;
"""


async def main() -> int:
    data = json.loads(DATA_PATH.read_text(encoding="utf-8"))
    assert len(data) == 1331, f"esperado 1331 rows, achei {len(data)}"
    for row in data:
        assert len(row["embedding"]) == 1024

    dsn = os.environ["DATABASE_URL"].replace("postgresql+asyncpg://", "postgresql://")
    conn = await asyncpg.connect(dsn)
    try:
        t0 = time.perf_counter()
        async with conn.transaction():
            for row in data:
                emb_str = "[" + ",".join(f"{x:.6f}" for x in row["embedding"]) + "]"
                await conn.execute(
                    UPSERT_SQL,
                    row["codigo"], row["secao"], row["divisao"], row["grupo"], row["classe"],
                    row["denominacao"], row["notas_explicativas"], row["exemplos_atividades"],
                    emb_str,
                )
        await conn.execute("ANALYZE cnae_taxonomy;")
        count = await conn.fetchval("SELECT COUNT(*) FROM cnae_taxonomy;")
        assert count == 1331, f"após load, count = {count}"
        print(f"Loaded {count} CNAE subclasses in {time.perf_counter() - t0:.2f}s")
        return 0
    finally:
        await conn.close()


if __name__ == "__main__":
    raise SystemExit(asyncio.run(main()))
```

> Variante de performance: trocar o loop por `conn.executemany(UPSERT_SQL, rows)` se 1331 inserts individuais demorarem (esperado <2s, mas medir).

## 8. Retrieval — `src/agente10/cnae/retrieval.py`

```python
"""Top-k retrieval of CNAE subclasses by embedding cosine similarity."""

from pydantic import BaseModel
from sqlalchemy import text
from sqlalchemy.ext.asyncio import AsyncSession


class CnaeCandidate(BaseModel):
    codigo: str
    denominacao: str
    similarity: float  # 1.0 = idêntico (cosine), 0.0 = ortogonal


_TOP_K_SQL = text(
    """
    SELECT codigo,
           denominacao,
           1 - (embedding <=> CAST(:emb AS vector)) AS similarity
    FROM cnae_taxonomy
    WHERE embedding IS NOT NULL
    ORDER BY embedding <=> CAST(:emb AS vector)
    LIMIT :k
    """
)


async def top_k_cnaes(
    db: AsyncSession,
    query_embedding: list[float],
    k: int = 10,
) -> list[CnaeCandidate]:
    """Return the k CNAE subclasses most similar to the query embedding.

    Uses pgvector's `<=>` cosine distance operator.
    """
    emb_str = "[" + ",".join(f"{x:.6f}" for x in query_embedding) + "]"
    result = await db.execute(_TOP_K_SQL, {"emb": emb_str, "k": k})
    return [
        CnaeCandidate(codigo=r.codigo, denominacao=r.denominacao, similarity=float(r.similarity))
        for r in result.all()
    ]
```

**Notas:**
- pgvector `<=>` é cosine **distance** (0 = idêntico). `similarity = 1 - distance` reverte para similarity (1 = idêntico).
- Passa embedding como string `"[a,b,c]"` com `CAST(... AS vector)` — formato aceito pelo pgvector e seguro contra problemas de adapter type entre asyncpg e SQLAlchemy.
- Não seta `ivfflat.probes`: default = 1 explora apenas 1 partição. Com 1331 rows e 40 lists (~33 rows/partição), recall pode cair. Vamos medir nos golden tests; se < 0.85, adicionar `SET LOCAL ivfflat.probes = 10;` antes do SELECT (a função pode encapsular ou deixar pro caller).

## 9. Voyage client — `src/agente10/integrations/voyage.py`

```python
"""Async wrapper around the Voyage AI SDK for embedding generation."""

import voyageai

from agente10.core.config import settings


class VoyageClient:
    """Thin async wrapper. Use embed_query for retrieval, embed_documents for indexing."""

    def __init__(self, api_key: str | None = None, model: str | None = None) -> None:
        self.client = voyageai.AsyncClient(api_key=api_key or settings.voyage_api_key)
        self.model = model or settings.voyage_model

    async def embed_query(self, text: str) -> list[float]:
        resp = await self.client.embed([text], model=self.model, input_type="query")
        return resp.embeddings[0]

    async def embed_documents(self, texts: list[str]) -> list[list[float]]:
        resp = await self.client.embed(texts, model=self.model, input_type="document")
        return resp.embeddings
```

**Config (`src/agente10/core/config.py`):**

```python
voyage_api_key: str | None = None
voyage_model: str = "voyage-3"
```

`voyage_api_key` é opcional para permitir testes sem a key (golden tests pulam, unit tests mockam). O construtor de `VoyageClient` propaga o erro do SDK se for chamado com `None`.

**Dependência:** `voyageai>=0.3` em `[project].dependencies` no `pyproject.toml`.

### 10.2 Teste `tests/integration/test_cnae_retrieval.py`

```python
"""Recall@5 of CNAE retrieval over a CSV fixture of representative spend descriptions."""

import csv
from pathlib import Path

import pytest

pytestmark = [pytest.mark.integration, pytest.mark.voyage]

GOLDEN_PATH = Path(__file__).parent.parent / "fixtures" / "cnae_golden.csv"
MIN_RECALL_AT_5 = 0.85


def _load_golden() -> list[dict]:
    with GOLDEN_PATH.open(encoding="utf-8") as f:
        return list(csv.DictReader(f))


@pytest.mark.asyncio
async def test_recall_at_5(db_session, voyage_client) -> None:
    from agente10.cnae.retrieval import top_k_cnaes

    rows = _load_golden()
    assert len(rows) >= 10, "fixture precisa de pelo menos 10 linhas"

    misses: list[tuple[str, str, list[str]]] = []
    for row in rows:
        emb = await voyage_client.embed_query(row["descricao"])
        candidates = await top_k_cnaes(db_session, emb, k=5)
        codigos = [c.codigo for c in candidates]
        if row["cnae_esperado"] not in codigos:
            misses.append((row["descricao"], row["cnae_esperado"], codigos))

    recall = 1 - (len(misses) / len(rows))
    if misses:
        miss_report = "\n".join(
            f"  - {d!r}: esperado {e}, top-5 = {c}" for d, e, c in misses
        )
        print(f"\nMisses ({len(misses)}/{len(rows)}):\n{miss_report}")
    assert recall >= MIN_RECALL_AT_5, f"recall@5 = {recall:.2f}, esperado ≥ {MIN_RECALL_AT_5}"
```

## 13. Riscos e mitigações

| Risco | Mitigação |
|---|---|
| XLS do IBGE muda estrutura/encoding entre versões | Parser tolerante a colunas extras/reordenadas; valida count == 1331 fail-hard; instrução em `data/cnae_2.3/README.md` para regenerar |
| Voyage API instável durante embed | Tenacity backoff exp em 429/5xx; embed_taxonomy.py escreve incremental e retoma do último codigo |
| Embeddings determinísticos? | Voyage-3 não é 100% determinístico (float drift entre runs); aceita-se — embedding bundled é "verdade canônica" daquele run, e golden tests provam que a qualidade é estável |
| ivfflat com poucos rows tem recall ruim | Mensurável nos golden tests; ajustar `probes` ou `lists` (incremental, sem migration nova: `ALTER INDEX … SET ...` ou recriar) |
| Golden tests com CNAEs errados (eu chutei alguns) | Task no plano: validar cada `cnae_esperado` contra `taxonomy.json` ANTES de gravar fixture |
| 7MB no repo via Git LFS necessário? | Não — Git lida bem com arquivos ≤100MB; LFS é overhead para benefício zero aqui |
| Pgvector + asyncpg type adapter quirks | Usar string-cast `CAST(:emb AS vector)` em vez de adapter; testado em pgvector 0.4.x |
| Re-embedar tudo no futuro (voyage-3.5) | Documentado: `uv run python scripts/embed_taxonomy.py --force` regera + commit + `make load-cnae` em cada env |
| Voyage SDK lança em runtime sem API key | `VoyageClient.__init__` aceita None mas SDK falha lazy; testes voyage skipam via marker; runtime production tem key obrigatória (Sprint 2 valida via settings) |
